“git命令小小笔记

Veröffentlicht am 2019-04-27

1、cd：目录 ，
cd：d:brog
就是进入到指定目录。
2、git init
新建repository仓库，初始化1个repository仓库，生成1个.git的隐藏文件夹。
3、npm install
安装组件。
4、git clone 拷贝git仓库到自己家里查看或修改。如：
git clone url 克隆该url的项目下载下来
下载next主题
git clone https://github.com/theme-next/hexo-theme-next themes/next
这个命令会生成在themes文件中，如果是其他的就会放在simplegit目录中。
5、ls：查看根目录的文件。
6、git status -s查看文件属性。
7、git add添加该文件并缓存格式如下：
git add things1.txt things2.txt
8、vim：修改指定文件内容
vim things1.txt

#wtf？
再使用git status -s查看属性
AM things1.txt
A things2.txt
AM是缓存的文件有所改动。
9、git diff：显示显示已写入缓存与已修改但尚未写入缓存的改动的区别
尚未缓存的改动：git diff
查看已缓存的改动： git diff –cached
查看已缓存的与未缓存的所有改动：git diff HEAD
显示摘要而非整个 diff：git diff –stat
git status -s只看的到改动属性和文件。而git diff可以看到缓存未缓存的内容改动。
10、git config设置信息
git config –global user.name “1479679118pjx”
git config –global user.email ”不告诉你.com“
11、git commit 将缓存内容放在仓库中，和add一般连用。
创好自己的username 和email等配置信息。如第10.
git add things1.txt things2.txt
git status -s查看。
git commit提交。
或者将git add的提交缓存省略
git commit -a
使用vim修改好内容后
接着再用
git commit -a
就能放到仓库啦
